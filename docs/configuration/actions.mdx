# Actions

Use `section_actions` to customize interactions per section. The card does not expose a global card action because it represents a single pool.

Supported actions are `more-info`, `navigate`, `url`, `call-service`, `toggle`, and `none`.

```yaml
section_actions:
  pump:
    tap_action:
      action: toggle // [!code highlight]
  temperature:
    tap_action:
      action: more-info // [!code highlight]
```

Default behavior is kept simple: gauges and mode open more-info by default, the pump toggles by default, and the other sections do nothing on hold or double tap unless you override them.
